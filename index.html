Implementation Details

We do everything post-RoPE, since these are inputs that actually directly enter attention / the dot product, even though the pre-RoPE K/Q's are more low-rank. RoPE shows up here as a rotation of the Q/K's around the PC1 axis in L0H2 (note the i-th RoPE pair is rotated by freq_i · pos_idx. For any references to key and queries, assume post rope.

For each layer, for each query head independently (keys shared within its GQA group), we find PC1, PC2, PC3 with:

def get_comps(Q, K):                      # Q, K: (seq_len, head_dim)
    unitQ = Q / (np.linalg.norm(Q, axis=1, keepdims=True) + 1e-12) # ||Q|| just acts as a temperature in the softmax
    X  = np.concatenate([K, unitQ], axis=0)
    Xc = X - X.mean(axis=0, keepdims=True)
    cov = (Xc.T @ Xc) / (Xc.shape[0] - 1)
    evals, evecs = np.linalg.eigh(cov)
    return evecs[:, ::-1][:, :3].T        # (3, head_dim)

For each query i we run regular (full-rank) attention, then keep the top-5 attended keys:

score_j   = <q_i, k_j> / sqrt(head_dim)     for all j <= i
attention = softmax(score)                    # over the sequence
top5      = np.argsort(attention)[::-1][:5]

Finally we project all vectors onto the PCA basis and draw them with Plotly. The projection is lossy (top-3 ≈ 50–65% of variance) and the top-5 come from the full-dimensional attention, so a highlighted key may not be the nearest one in the 3-D view, although under reset view, we see they approximately are.
